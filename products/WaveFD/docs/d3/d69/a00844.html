◆ visualize_fps

unsigned int wavefd::WaveFDData::visualize_fps

Determines the inner time interval of visualized frames.

This is not the real fps in the real world.

If 96000 samples simulation cost 180 seconds and the WaveFDData::sample_rate = 48000.

In the case WaveFDData::visualize_fps = 60.

Your window will update every (48000/60) * (180/96000) = 1.5 seconds.

Each window update will skip 48000/60 = 800 samples.

If you want to show every sound field result, you should set wavefd::VISUALIZATION_FULL to WaveFDData::visualization_flag

◆ sample_rate

unsigned int wavefd::WaveFDData::sample_rate

This will be calculated by WaveFD, and it is based on the width and height you have sent to wavefd::create.

This result is also based on CFL condition.

It means the minimum sample rate we can set to make sure the accuracy of the simulation.

You are not supposed to change it.

Will be assigned after calling wavefd::create.

Will be reset after calling wavefd::dispose.
